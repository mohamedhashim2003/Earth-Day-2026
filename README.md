# 🌍 Virtual Energy Kiosk

**Galala University Earth Day Ideas Competition 2026 Submission**

An interactive, gamified web platform designed to raise community awareness about energy conservation by translating abstract electrical consumption into relatable, real-world metrics.

## 🚀 Project Overview

The Virtual Energy Kiosk bridges the gap between theoretical engineering knowledge and daily behavioral changes. Instead of a static presentation, this application serves as a dynamic "digital twin" of everyday household and campus devices. Users can interactively select devices to instantly calculate their hidden energy draw.

By translating raw wattage into tangible equivalents—like the cost of campus coffee or the distance driven in an EV—the platform simplifies energy concepts for the general public and encourages more efficient habits.

## ✨ Features

* **Interactive Simulation:** Click to "plug in" common electronics (e.g., Phone Chargers, Space Heaters, Desktop PCs).
* **Gamified Metrics:** Automatically calculates yearly energy draw (kWh) based on 8 hours of daily use/standby.
* **Relatable Equivalents:** Translates kWh into relatable metrics:
    * Equivalent cups of coffee.
    * Kilometers driven in an electric vehicle.
* **Two-View Architecture:** Seamless transition between the project mission statement and the interactive dashboard.
* **Fully Responsive:** Clean, modern UI designed with a green "eco" theme.

## 🛠️ Technology Stack

* **Frontend:** HTML5, CSS3 (Vanilla)
* **Logic:** JavaScript (Vanilla/ES6)
* **Hosting:** Fully static, designed for instant deployment on GitHub Pages.

## ⚙️ Quick Start & Deployment

Since the application is built entirely on client-side technologies, no local server is required to test it.

1.  Clone the repository:
    ```bash
    git clone https://github.com/mohamedhashim2003/Earth-Day-2026.git
    ```
2.  Open `index.html` in any modern web browser to view the application locally.
3.  **To Deploy:** Go to your repository settings on GitHub, navigate to **Pages**, and set the source to deploy from the `main` branch.
